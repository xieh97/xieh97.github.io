Label Studio 是一个前后端分离的项目，后端是 Django，前端是 React 应用 。

下面是完整的本地开发环境搭建步骤：

~~~text
本文使用到的工具：
- Python 3.13（下载地址：https://www.python.org/downloads）
- Yarn 1.22.22（下载地址：https://github.com/yarnpkg/yarn）
- NodeJS v24.13.1（下载地址：https://nodejs.org/en/download）
- Git for Windows 2.53.0（下载地址：https://git-scm.com/install/windows）
~~~

## 📥 1. 获取源码

~~~bash
# 克隆仓库（使用 --depth 1 可以加快下载速度）
git clone https://github.com/HumanSignal/label-studio.git --depth 1
cd label-studio
~~~

## 🐍 2. 配置后端 Python 环境

~~~bash
# 安装 Poetry（Python 依赖管理工具）
pip install poetry

# 查看 Poetry 的环境配置
poetry env info

# 配置 Poetry 将虚拟环境创建在项目目录内（方便管理）
poetry config virtualenvs.in-project true

# 安装所有 Python 依赖
poetry install
~~~

## 🗄️ 3. 初始化数据库

~~~bash
# 执行数据库迁移
poetry run python label_studio/manage.py migrate

# 收集静态文件（但此时前端资源还没构建，后面还要再收集一次）
poetry run python label_studio/manage.py collectstatic --noinput
~~~

## ⚛️ 4. 构建前端资源（关键步骤！）

~~~bash
# 进入前端目录
cd web

# 配置 yarn 使用 Git Bash（for Windows）
yarn config set script-shell "~/Git/bin/bash.exe"

# 安装前端依赖（使用 yarn）
yarn install --frozen-lockfile

# 构建前端资源
yarn build

# 返回项目根目录
cd ..
~~~

构建完成后，前端文件会生成在 `web/dist/apps/labelstudio` 目录下，这是 Django 静态文件收集时要找的路径。

## 🔄 5. 再次收集静态文件

~~~bash
# 现在 Django 可以找到前端资源了
poetry run python label_studio/manage.py collectstatic --noinput
~~~

## 🚀 6. 启动开发服务器

~~~bash
# 启动后端 Django 服务
poetry run python label_studio/manage.py runserver
~~~

访问 http://localhost:8080 即可看到 Label Studio 界面 。

## ⚡ 7. 开启热重载（开发体验优化）

如果希望修改代码后浏览器自动刷新，可以配置热重载（HMR）。

首先，在项目根目录创建 `.env` 文件：

~~~bash
# 启用热重载
FRONTEND_HMR=true
FRONTEND_HOSTNAME=http://localhost:8010
DJANGO_HOSTNAME=http://localhost:8080
~~~

然后，重新打开两个终端，分别启动前后端：

~~~bash
# 终端1: 启动 Django 后端
poetry run python label_studio/manage.py runserver

# 终端2: 启动前端开发服务器
cd web && yarn dev
~~~

此时访问 http://localhost:8080，前端会通过 8010 端口提供热重载服务 。

## ⚠️ Windows 特别提醒

- **路径问题**：如果在安装依赖时遇到 `lxml` 安装失败，可以下载预编译的 `.whl` 文件手动安装
- **Node.js 版本**：建议使用 Node.js 16+
- **如果构建失败**：可以尝试删除 `node_modules` 重新安装：
  ~~~bash
  cd web
  rm -rf node_modules  # Windows 用 rmdir /s node_modules
  yarn install
  ~~~

按照这个流程操作，Label Studio 本地开发环境就能正常运行了。
